# Економічний аудит

## Контекст
Cimeika поєднує SPA-шари (CimeikaReact, маркетинговий `cimeika`), API-ядро на Hugging Face Space (`cimeika-api`), знаннєву базу `ciwiki`, інфраструктурні конфіги та спеціальні модулі на кшталт цього real-time сервісу. Ці категорії визначають, де створюється цінність та як її монетизувати.

## Категорійний огляд
- **App/Web**: CimeikaReact як головний фронт (чат Ci, Подія/Настрій/Маля/Казкар, галерея, календар), маркетинговий сайт `cimeika` для лідів і довіри.
- **API/Core**: HF Space `cimeika-api` + опційний дзеркальний репозиторій; оркестрація чатів, подій, маршрутизації персон та реального часу.
- **Docs/Knowledge**: `ciwiki` як єдине текстове сховище й API `/api/v1/ciwiki/content` для вбудовування у SPA та Space.
- **Infra/Ops**: `Cimeika_Project_Structure` для шаблонів серверів/бекапів/моніторингу та `workflow-` для automation.
- **Special modules**: `cimeika-real-time-data-app` для візуалізації/реального часу, `album_ai` та `MrijkiMarijki` як допоміжні медіа/дитячі сервіси.

## Економічний потенціал та моделі
- **Core subscription (B2C/B2B2C)**: базова безкоштовна точка входу (чат Ci + обмежений обсяг Подія/Настрій) з платними пакетами за глибину сценаріїв, історію подій, приватні галереї й часозони календаря.
- **API-ліцензування**: платні плани за доступ до `/api/ci`, `/api/gallery`, `/api/calendar` і realtime-стрімів; тарифи за запити й SLA, white-label для партнерів (освіта/виховання/астрологія).
- **Платні add-ons**: динамічні історії Казкаря, творчі сеанси Маля, AI-підсилені галереї (генеративні прев’ю), розширені «Подія» наперед (пам’ятки, інтеграція з зовнішніми календарями).
- **Контент та знання**: ciwiki як «pro library» з платним офлайн-доступом, експертними пресетами та експортуванням легенд/шляхів у PDF/JSON.
- **Маркетинговий upsell**: `cimeika`-лендінг для захоплення лідів → перенаправлення у SPA та Space з персональними тригерами (перший ритуал, перша подія, перше збереження у галереї).

## Оптимізація витрат і доходів
- **Короткий горизонт (0–4 тижні)**
  - Зашити тарифні прапори у конфіг SPA (`frontend/src/core/config`) і FastAPI/Node API, щоб розмежувати free/pro поведінку без форку коду.
  - Впровадити ліміти запитів і тайм-аути в `/api/ci` та галереї/календарі для контролю витрат на HF/OpenAI та файлове I/O.
  - Додати CTA/маршрут з реального часу (`/realtime`) до CimeikaReact для перетоку безкоштовних користувачів у платні флоу.
  - Відокремити білінгові події (створення події, збереження зображення, AI-запит) у логах для подальшого скорингу LTV.
- **Середній горизонт (1–2 квартали)**
  - Підготувати white-label профілі (брендинг, API ключі, домен) на рівні `cimeika-api` та `CimeikaReact` для партнерських рішень.
  - Об’єднати ciwiki + SPA у єдиний пакет контенту: динамічне підвантаження легенд/гайдів у модулі Казкар/Маля з платними сценаріями.
  - Оптимізувати інфраструктуру: кеш realpath/JSON (галерея) уже є — поширити на календар/legend, винести статику на CDN, розділити білди для маркетингу та SPA.
  - Формалізувати метрики product/finance: активація (перша подія + перше зображення), ARPU/MAU, розмір кошика add-ons, вартість інференсу на користувача.

## Метрики й експерименти (пропозиція)
- A/B: CTA до `/realtime` vs. прямий чат Ci; метрика — конверсія у створення події.
- Paywall-тест: галерея >N елементів або Казкар>3 історії → пропозиція pro; метрика — апгрейд та відтік.
- Цінова чутливість: пакети «family», «mentor», «school» з різними квотами `/api/ci` та медіа-сховища.
- Інфраструктурна економія: виміряти час відповіді та I/O на галереї/календарі до/після кешів; ціль — -20% latency і -15% CPU/IOps.
